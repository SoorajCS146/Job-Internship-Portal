1. Project Context

This project is a backend-focused microservices demo implemented using Spring Boot.

Backend Architecture
* Services
	- student-service ‚Üí port 9001
	- company-service ‚Üí port 9002
	- placement-service ‚Üí port 9003

* Infrastructure
	eureka-server ‚Üí port 8761
	api-gateway (gateway-server) ‚Üí port 9000

* Service Discovery: Eureka is used by backend services
* Database: H2 (file-based persistence)

* API Access Rule: 
	- Frontend must primarily communicate via API Gateway: http://localhost:9000
	- Direct service access is allowed ONLY as a fallback (for demo resilience when gateway is unavailable)

8. Shared API Client with Fallback (apiClient.js)

All HTTP communication MUST go through a single helper.

Responsibilities :
	- Try API Gateway first

	- If gateway fails:

		- Retry using direct service URL

		- Log warning to console

	- Throw error only if BOTH attempts fail

Required Function: async function apiFetch(serviceName, endpoint = "", options = {})

Behavior:-
	a) Construct Gateway URL
	b) Call Gateway
	c) If failed:
		- Retry using fallback URL
	d) Return parsed JSON response

Service-specific JS files MUST NOT call fetch() directly.
